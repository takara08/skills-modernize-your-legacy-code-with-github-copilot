# 学生口座管理システム テスト計画

このテスト計画は、現行 COBOL アプリケーションのビジネスロジックを網羅的に検証するためのものです。Node.js への移行後も同様のテストケースを利用できます。

| Test Case ID | Test Case Description | Pre-conditions | Test Steps | Expected Result | Actual Result | Status (Pass/Fail) | Comments |
|--------------|----------------------|----------------|------------|-----------------|---------------|--------------------|----------|
| TC-01 | 残高照会（初期値） | 初期残高が 1000.00 であること | 1. アプリを起動<br>2. メニューで「1. View Balance」を選択 | 残高が「001000.00」と表示される |  |  |  |
| TC-02 | クレジット処理（正常系） | 初期残高が 1000.00 であること | 1. アプリを起動<br>2. メニューで「2. Credit Account」を選択<br>3. 金額「500」を入力 | 新しい残高が「001500.00」と表示される |  |  |  |
| TC-03 | デビット処理（正常系） | 初期残高が 1000.00 であること | 1. アプリを起動<br>2. メニューで「3. Debit Account」を選択<br>3. 金額「200」を入力 | 新しい残高が「000800.00」と表示される |  |  |  |
| TC-04 | デビット処理（残高不足） | 初期残高が 1000.00 であること | 1. アプリを起動<br>2. メニューで「3. Debit Account」を選択<br>3. 金額「2000」を入力 | 「Insufficient funds for this debit.」と表示され、残高は変わらない |  |  |  |
| TC-05 | クレジット後の残高照会 | クレジット処理後 | 1. クレジット処理で「500」を追加<br>2. メニューで「1. View Balance」を選択 | 残高が「001500.00」と表示される |  |  |  |
| TC-06 | デビット後の残高照会 | デビット処理後 | 1. デビット処理で「200」を減算<br>2. メニューで「1. View Balance」を選択 | 残高が「000800.00」と表示される |  |  |  |
| TC-07 | 無効なメニュー選択 | なし | 1. アプリを起動<br>2. メニューで「5」など無効な値を入力 | 「Invalid choice, please select 1-4.」と表示される |  |  |  |
| TC-08 | 終了処理 | なし | 1. アプリを起動<br>2. メニューで「4. Exit」を選択 | 「Exiting the program. Goodbye!」と表示され、プログラムが終了する |  |  |  |
| TC-09 | クレジット金額が0 | なし | 1. アプリを起動<br>2. メニューで「2. Credit Account」を選択<br>3. 金額「0」を入力 | 残高は変わらず「001000.00」と表示される |  |  |  |
| TC-10 | デビット金額が0 | なし | 1. アプリを起動<br>2. メニューで「3. Debit Account」を選択<br>3. 金額「0」を入力 | 残高は変わらず「001000.00」と表示される |  |  |  |

> **備考:**
> - 「Actual Result」「Status」「Comments」はテスト実施時に記入してください。
> - 金額入力時のフォーマットやエラー処理は COBOL 実装に準拠しています。
